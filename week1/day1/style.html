TRANSCRIPTION

It’s time to flaunt your inner designer because we’re about to learn some CSS.

To start off I want to show you this website; if you look at it,

well it’s quite quite pretty.

Look at that; that’s really nice.

Now hopefully in the future we can create something like this.

But I want to show you what this website will look like, without CSS.

Here it is.

Pretty impressive, right?

With just CSS you’ll be able to create something like this.

So let’s get started, shall we?

I’ve created here for us a simple website – nothing that we haven’t seen before; you can see that I

have created semantic tags with ‘header’ tag, ‘nav’ tag, an unordered list [ul] that has some links, an ‘h2’ tag and a ‘p’ tag that’s

wrapped in another semantic tag called ‘section’.

OK.

So again nothing crazy; if I go to the ‘About’ page it’s the exact same thing except for the About ‘li’

doesn’t have an anchor tag.

And then the same with ‘Login’.

OK so let’s talk about CSS. The way that we would add CSS to something like this is … well we would create a

CSS file.

So let’s do that.

I’m going to create a new file that I’m going to save.

Let’s call it ‘style.css’.

If I save that you’ll see at the bottom over here that Sublime Text now has CSS as the file type.

The rules with CSS are

very simple.

We essentially have a selector and then we have the squiggly brackets and within it we put our property,

colon and then value, and then we have this special character as well [semi-colon] at the end.

And and that is pretty much CSS. You know this, and you can pretty much write CSS – just like HTML, just a couple

of syntactic rules.

And once you get used to it, it’s very very simple.

But obviously this doesn’t really mean anything so let’s write actual code here.

I’m going to comment that out by again doing ‘Command /’ in Sublime Text.

You can see that the commenting is different than in HTML.

And let’s say that we want to make this ‘Home’ text red – the way that we would do it is – we want to select what’s

called the selector.

So here we want to say that the ‘h2’, we want that to be red.

So all I would say is: ‘h2’, curly brackets.

And then within here I want to say the property; I want to change the color to the value that is red.

So if I say that, and refresh, well, nothing happens, and that’s obvious, right? Because how does this file

know that this file exists?

We have to link it somehow.

And we do that in the ‘head’ and we haven’t talked too much about the ‘head’ tag element in the HTML section,

but by now you can probably tell that ‘head’ tag just contains information about what the page needs.

Maybe not the stuff that we see on a web page other than the title, but ‘meta’ tags and information that

we need for the website. OK.

So in here we can actually link to this stylesheet that we call ‘style.css’ with a ‘link’ tag.

This ‘link’ tag allows us to link this HTML file to something else.

And again Sublime Text has something very useful here.

If I hit ‘Tab’, look at that: it auto-fills it for me.

These are the standard attributes that you’ll see to link to CSS.

So you just have to get used to it and eventually it’ll become second nature.

So we have ‘rel’ here, which is the added attribute for ‘relation’ it just specifies what relationship

this link will have, and we have the default stylesheet and then for the type,

this is the media type, we want to say that it’s a text CSS file.

We’re essentially telling the HTML, “Hey just a heads up – we’re linking to a CSS file here”. And then the ‘href’ –

this should be familiar. We’re going to link to ‘style.css’.

So if I type in ‘style.css’ and save, let’s refresh, turns red.

And just to let you know that ‘index.html’ is in the same folder as ‘style.css’ – we can check

it over here.

If I click on Exercise, you see that yup, I have ‘style.css’ with ‘index.html’.

Perfect.

OK.

Now what if we also want to make the paragraph over here a separate color as well.

Well again we would do the selector and then we’ll do color … let’s do ‘pink’.

I save, refresh, and it’s pink. Hopefully you can see that. You also see here that Sublime Text does some really

nice coloring for us.

So if for example I put a property that doesn’t exist like a misspelled color, well you see that it

doesn’t highlighted properly with the blue color?

And you know that you’ve made a mistake. Again,

another great feature of Sublime Text.

So ‘CSS’ – what does it actually stand for?

It stands for Cascading Style Sheets, and ‘style sheets’ – that makes sense – that we’re creating

style sheets, but what does ‘cascading’ mean?

Well cascading means that it always takes the selector that is at the end.

And that’s simplified terms.

But I want to demonstrate for you what that means.

So let’s say at the bottom over here I do this again – paragraph – I want ‘color’ to be green. Well if I save

this what do you think will happen?

Let’s refresh and see.

It turns green.

And why is that? Well because it’s cascading.

It’s saying, “Okay I’ll set ‘h2’ to red.”

OK.

“I’ll set ‘paragraph’ to pink”, and then it says, “OK I’ll set ‘p’ to green”. So you can see here,

as it trickles down and cascades it takes the last input that it receives for a specific selector.

We’ll get more into the cascading effects of CSS in a later video.

But I want you to understand the basic concept around it.

OK.

I also wanted to show you something cool that you can do with Chrome.

And we did this in the first couple of video series but now you’ll actually understand what it means.

So again ‘View’, ‘Developer’, ‘Developer Tools’ and from now on I’ll be using this shortcut to open it, and you’ll

be able to do it as well.

And if you remember we have our style tab, and if we click on the style tab we have the HTML on the left

style’s over here and nothing for the styles.

But if I now go into my Home and I click on that, it will show you ‘color: red’.

So Google Chrome allows us to see our CSS styles so I can even add stuff here and I could say ‘background-color’

and let’s do black. Look at that.

So this is a really good way to play with your website and add colors.

You can check and uncheck here and also if you see something on a website that you really really like

you can go to that element.

Let’s try it off here.

Let’s go to

waitbutwhy.com.

And let’s say that we really really want to know how this was made.

Again right click, do ‘Inspect’ - it’ll take us exactly to that element and then we can see here some of the

properties, the CSS properties, that they’ve used. And the same here – you can see that the background color

is this and so on and so forth.

OK let’s close that for now.

The last thing I want to leave you with is, what if – just close this – what if I also want the About page to

have styles and the same with the Login page. As you can see, there’s no style for these ones.

But we want the same thing for the About and Login page.

Well as you can imagine we copy the ‘link’ tag and we go to About; we link it on to here, and then Login,

we can again; we’ll link it here.

If I save both of these files,

and now if I refresh, and go About – look at that – we have our style, and you may be wondering well what

if I want different styles, maybe for the Login.

I want this to be blue instead.

Well we can actually create multiple style sheets so I can say for example let’s create a new file and

we’ll call it ‘style2.css’.

I save that and this time I’m going to say that I want my ‘h2’ to have the color blue.

I save that; I go into my ‘login.html’;

And now I’ll make Login reference the second stylesheet instead of the first one.

So now if I refresh, Login is referencing our ‘style2.css’,

But all the other ones are still referencing the old one.

Ok I lied.

There is one thing that I want to add and that is that this is the common way of adding styles.

But there is actually two other ways that you won’t see as often especially when you’re starting off,

but

I wanted to show it to you so that you’re not surprised.

The first way of doing it is something called ‘inline styles’ and this was the original way that we incorporated

styles into our HTML and that is by specifying within the element, let’s say, let’s go with ‘header’ tag

over here.

We want to say style.

So it has a style attribute.

And then within here we can do ‘background-color’ – let’s do green, and refresh.

And we’re at the About page right now; we changed that for ‘index.html’.

So let’s go over here and look at that.

We have a green background.

And if you want to add more, again we do ‘color: red’ with save and refresh, and look at that – the header

has the color red and you’ll see here that these ones are still purple because they are ‘link’ tags

and they have the default linking.

OK.

So that’s one way and the third and last way is in the ‘head’ tag.

You can actually create ‘style’ tags, and then within these ‘style’ tags you can enter CSS.

So if we just refresh here,

so we have a fresh clean slate,

I can now say all the ‘p’ tags, or let’s do ‘li’ this time.

So all the lists; I want the lists to have ‘background-color’ of purple; so if I now save this and refresh.

Everything is purple, and that looks pretty yucky.

You can keep adding stuff on here and I can say ‘color: white’, save, refresh and it changed the color to white.

Now you may be wondering why don’t we just do it like this.

This seems a lot simpler than linking to another CSS file.

Well the reason is that for this website everything is fairly small.

It’s just a couple lines of CSS, but you can imagine with websites, as they get bigger and bigger you’ll

have hundreds and hundreds of lines of CSS, and having one massive file will be very very challenging

to maintain.

You can also think of it as if you were working on an HTML file, and you also had a team of designers

that were working on a CSS file.

You want to have a separation where you have some team members working on the CSS and then some team

members working on the HTML and that is a concept called ‘separation of concerns’.

It’s a fancy way of saying that everybody just worry about what they’re good at.

So, HTML, just worry about text, and CSS, just worry about styles.

This phrase ‘separation of concerns’ is something that we’ll encounter throughout later videos when we

get into JavaScript and React and it’s a very very important concept in web development and computer

science in general so we will get back to that,

but you know that it’ll be a phrase that will keep coming up.

So for now let’s delete this; let’s keep things the way that we have it.

Let’s refresh and in the next video I’m going to show you a couple of CSS properties that are more

exciting than just color and background.

I’ll see you in that one. Bye-bye
